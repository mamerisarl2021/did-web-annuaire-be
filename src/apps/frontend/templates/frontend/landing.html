Built for trust

Everything you need to manage decentralized identities

W3C Compliant

DID documents are created following the W3C DID Core specification. Each published document carries a Data Integrity proof using the ecdsa-jcs-2019 cryptosuite, ensuring that the content is tamper-evident and cryptographically verifiable by any conformant resolver.

Cryptographic Signing

Documents are signed using ECDSA P-256 through Keyfactor SignServer. Signing keys are generated and managed in a dedicated crypto token, and every signature is logged for auditability.

Universal Resolution

Resolve did:web identifiers through the integrated DIF Universal Resolver. Published DID documents are served as static JSON files at their spec-defined HTTPS paths, making them accessible to any standards-compliant verifier or wallet.
